Agentic Workflow Automation

Autonomous Email Agent for Credentialing and Roster Automation

September 2025 to Present Healthcare operations

Designed an autonomous email agent for credentialing and roster automation across email-driven workflows, including DEA verification, medical license validation, and malpractice insurance tracking.

Problem

High-friction workflows hidden inside email threads

Credentialing workflows were spread across long email threads and required repeated handling of verification, checklist tracking, and compliance-sensitive follow-ups.

Solution

Thread-aware multi-agent orchestration

Architected a multi-agent system using LangGraph with email-thread-aware finite state machines and specialized agents for intent classification, entity extraction, checklist tracking, and action planning.

Impact

Operational gains with explicit guardrails

Reduced processing time by 75% across 1,500+ cases, with a 92% task completion rate and less than 5% escalation.

Architecture

  • Email-thread-aware finite state machines with rollback capabilities for error recovery.
  • Specialized agents for request, remind, clarify, and escalate actions.
  • RAG-enhanced retrieval over historical cases and policy documents.
  • Dynamic email templating, audit logging, and human-in-the-loop gates.

Modeling

  • Schema-constrained generation with Qwen 3 8B JSON mode and Pydantic validators.
  • 96% field extraction accuracy with automatic retry logic and confidence-based recovery.
  • Adaptive routing using LLM-as-judge between zero-shot, few-shot, and fine-tuned models.
  • RabbitMQ message queuing with exponential backoff retries and dead-letter exchanges.

Stack

Qwen LangChain LangGraph FastAPI PostgreSQL Redis RabbitMQ

Hybrid RAG and Knowledge Graphs

Provider Contracts AI and Intelligent Query Tool

June 2024 to September 2025 Contract intelligence

Developed an end-to-end hybrid RAG contract intelligence system to query structured entities from scanned PDFs, including TIN, effective dates, renewal terms, and payment schedules.

Problem

Scanned contracts were searchable but not queryable

Contract intelligence required extracting structured entities from scanned documents and supporting auditable queries over large volumes of provider contracts.

Solution

Multistage extraction plus hybrid retrieval

Combined OCR, layout parsing, Qwen fine-tuning, multimodal extraction, and a three-way hybrid retrieval system covering graph traversal, dense search, and sparse search.

Impact

Auditable answers over 100k+ contracts

Achieved 94% extraction accuracy, improved downstream extraction quality by 12%, and supported sub-second latency for contract analytics queries.

Pipeline

  • Textract OCR for scanned PDFs followed by LayoutLMv3-based region parsing.
  • LoRA fine-tuning of Qwen 3 8B on contract entity extraction and structured decoding.
  • Qwen 2.5-VL for multimodal extraction from provider profile screenshots.
  • Natural-language query interface with LLM-powered routing and text-to-Cypher translation.

Retrieval Design

  • Neo4j knowledge graph storing entities, relationships, and hierarchies across 100k+ contracts.
  • Cypher queries for relationship-based reasoning.
  • Dense vector search and BM25 sparse retrieval with reciprocal rank fusion.
  • Provenance tracking at document, page, and clause level for audit compliance.

Stack

Qwen 3 Qwen 2.5-VL LoRA FAISS Neo4j Document AI FastAPI

Clinical NLP

CodEx Clinical Entity Extraction and Automated Medical Coding

June 2023 to May 2024 Clinical coding

Developed an end-to-end clinical NLP pipeline automating ICD-10 and CPT code assignment from medical charts across healthcare facilities.

Problem

Medical coding needed precision at scale

Manual coding was time-intensive and required reliable extraction, normalization, and retrieval across a large medical code space.

Solution

Layered NLP pipeline rather than a single model

Combined BERT-CRF entity extraction, BERT QA, Clinical T5 normalization, and semantic search over 900k codes with reranking for final recommendation quality.

Impact

Large throughput with measurable gains

Reduced manual coding time by 60%, achieved 0.91 F1 for clinical entity extraction, and handled 2,000+ requests per day in production.

Model Components

  • BERT-CRF for entity extraction across conditions, procedures, labs, and medications.
  • Multi-label enrichment for assertion, temporality, and subject.
  • BERT QA for extracting lab values, dosages, and dates.
  • Clinical T5 fine-tuned on 800k entity-code pairs for terminology normalization.

Retrieval and Deployment

  • Bi-encoder plus ScaNN semantic search across 900k codes with 18 ms latency and 95% recall@20.
  • Cross-encoder reranking improved precision by 13 percentage points.
  • PySpark pipelines handled data extraction and offline processing.
  • Deployment on AWS EKS with continuous monitoring for latency and quality.

Stack

BERT-CRF T5 ScaNN PySpark FastAPI Docker AWS EKS

Personal Project

Research Paper Assistant

Independent build Research workflows

Built a FastAPI application that imports arXiv papers by title or link, downloads PDFs, reads them in-app, lets users save highlights, and supports grounded question answering over the paper contents with OpenAI.

Problem

Research reading tools rarely stay grounded in the paper

I wanted a workflow where paper reading, note-taking, and question answering happen in one place with answers tied back to extracted paper context instead of loose chat history.

Solution

Paper workspace with retrieval-backed and agentic chat

The app imports arXiv papers, chunks and embeds extracted PDF text, supports context selection and persistent highlights, and exposes two chat modes: standard retrieval-backed Q&A and an agentic LangGraph workflow.

Product Shape

Designed like a focused single-paper research console

Users can import a paper, read the PDF inline, save highlights and notes, and ask questions with context grounded in selected chunks, retrieved chunks, and prior memory.

Core Flow

  • Imports a paper from an arXiv title or direct arXiv link.
  • Downloads the PDF, extracts text with `pypdf`, and chunks it for retrieval.
  • Stores papers, chunks, highlights, and chat history in a SQL-backed app state.
  • Serves the original PDF inline through FastAPI for in-app reading.

Chat and Agentic Mode

  • `Standard` mode retrieves relevant chunks and generates answers grounded in paper context.
  • `Agentic` mode uses LangGraph to plan the question, retrieve context in one or two passes, and synthesize the answer.
  • Selection-aware chat combines chosen chunks, retrieved chunks, and saved highlights memory.
  • Responses carry cited chunk ids so answers stay tied to extracted content.
